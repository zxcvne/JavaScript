## component && module

```
    컴포넌트 : 프로젝트 개발할때 기능별로 나눠서 반복적으로
            사용할 수 있는 웹 애플리케이션을 기능별로 나눈 독립적인 부분

    모듈 : 재사용 가능한 코드의 집합으로, 특정 기능을 수행하는 코드를 하나의 단위로 묶어 관리

    "컴포넌트"는 주로 사용자 인터페이스(UI)를 구성하는 독립적이고
    재사용 가능한 부분을 가리키며, 재사용 가능한 중복되는 기능의 반복은
    "모듈"로 묶어서 관리 할 수 있다.


    장점 : 중복 코드 작성하지 않고 재사용 가능
          기능과 구조를 명확하게 분리 가능
          독립적인 요소 -> 유지보수 편리

    컴포넌트와 모듈 단위로 코드를 나누면 코드를 깔끔하게 유지할 수 있고 코드를 쉽게 관리할 수 있다.
    모듈은 독립되어있지만 필요할때 코드를 조합하여 사용.
```

## 모듈 시스템

```
    ECMA Script Module System

    module : 하나의 파일을 의미
    module들은 자신만의 스코프를 가짐
    module 내부에 작성된 기능들은 서로 외부에 공유할 수 없고 모듈 내부에서만 사용 가능함 // 식별자들이 모두 비공개 처리

    export 키워드 : 식별자의 선언문 앞에 export 키워드를 활용하면 다른 파일에서 사용이 가능하다.
    import 키워드 : export로 선언된 공개된 모듈을 가져올 수 있다.
```
